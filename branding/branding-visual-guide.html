01 — Visual north star

What this brand must feel like

The brand should feel like: A skilled craftsperson you'd trust to build something important for your business — warm, direct, and present from first call to delivery.

Not like: A faceless agency portal, an AI vendor pitch, or a "solutions" website.
One-line visual thesis

A solo craftsperson's studio — warm stone and pine, editorial presence, and the specific quiet confidence of someone who signs their own work.

02 — Direction mix

Editorial / Human leads

Editorial / Human — 65% Enterprise / Stable — 35%

The visual system leads with warmth and human presence: stone-paper backgrounds, editorial serif moments, founder portrait, first-person voice. The Enterprise/Stable layer provides structure: pine anchor, clean grid, proof modules that signal reliability without over-claiming.

What this mix prevents

  • Any Cutting-edge / Innovative influence — creates distance from the target audience
  • 50/50 balance — hedging produces visual blandness
  • Agency authority signals (awards, partner logos, team headshots)
  • SaaS product visual language (dashboards, feature grids, pricing tables)

  • Purple, blue, or teal in any component
  • A second accent color alongside terracotta
  • Large fields of terracotta (it's a spice, not a base)
  • Tech gradients on any primary surface

07 — Proof presentation

Honest, specific, structured to grow

High trust burden. Zero third-party reviews at launch. Every proof decision must be honest and build toward the moment real reviews arrive.

Phase 1 — now (zero reviews)

  • Founder portrait + proof chips as illustrative outcomes
  • Credential bar: 6 years engineering, one-person promise, 2-day audit
  • Before/after: Quán Phở — named, specific, no fabricated metrics
  • Testimonial card structure with honest placeholder framing
  • Free audit offer as the proof-of-quality CTA

Phase 2 — first reviews arrive

  • Replace placeholder with real testimonial: name + photo + verbatim quote
  • Remove "in progress" framing entirely
  • Add GBP / Google review link once live
  • Phase 3: Add outcome metrics to before/after when real and specific

What must never be fabricated

  • Review counts or star ratings without a verified platform source
  • Client revenue numbers or conversion rate improvements
  • Named client results without explicit written permission
  • "AI transformation" before/after metrics
  • Any testimonial without a real name and real photo

11 — Do / Don't rules

Blunt. Visual. Operational.

Do

  • Lead every section with the problem or cost of inaction — solution second
  • Use first-person singular in all copy — "I build," "I send back," "you reach me directly"
  • Keep Thao's portrait visible in the hero above the fold on all viewports
  • Use pine as the trust anchor — never introduce a second color
  • Alternate section backgrounds so no two adjacent bands share a shade
  • Limit terracotta to the single most important CTA per page
  • Use sentence case for all headings without exception
  • Use the double-shell card treatment for all service and proof cards
  • Keep the Cormorant Garamond italic to one fragment per section — rare by design
  • Place the free audit CTA in the hero and the closing section at minimum

Don't

  • Use stock photography — laptops, handshakes, smiling-at-screen
  • Introduce any second accent color for any reason
  • Show workflow diagrams, node maps, automation flows, or dashboard screenshots
  • Write in plural voice anywhere on the site
  • Display a testimonial or star rating without a real name and real photo
  • Use pure #FFF as any background — warm stone only
  • Use Cormorant Garamond for body, UI, nav, or caption text
  • Fabricate metrics, review counts, or "AI transformation" results
  • Use hard offset block shadows — soft pine-tinted depth only
  • Use any color outside the locked palette
  • Use Title Case for any heading

13 — Implementation notes

What must be preserved. What can flex.

Must preserve exactly

  • Token names and hex values — reference via CSS custom property, never hardcoded hex
  • Double-shell card structure — outer shell + inner core + inset highlight, in that order
  • Island button — pill wrapper, inner icon wrapper, magnetic hover
  • Section background sequence — never two adjacent sections the same shade
  • Reduced-motion guard on every animation
  • Sentence case on all headings
  • Cormorant Garamond italic at display size only — one fragment per section

Accessibility

  • All interactive elements: :focus-visible ring using --pine
  • Color never the only information carrier — pair with shape, weight, or label
  • All images: descriptive alt text. Portrait: "Thao Phuong, founder of TechPersona Studio"
  • Form labels: visible, never placeholder-only
  • Reduced motion: required on all animations
  • Minimum tap target: 44×44px on all buttons
  • Body contrast (--ink on --paper): ~14:1 — passes AAA

Future agents must never improvise

  • A second accent color
  • A new typeface
  • Gradient or glassmorphism on any surface
  • Plural voice in copy
  • Hard offset block shadows
